- **사용자 프로필 필수/선택 속성 추가**
  - **무엇을**: `User` 엔티티에 연령대(`AgeGroup`)와 한줄 소개(`Introduction`) 필드를 추가했습니다.
  - **왜**: 서비스 기획에 따라 사용자의 연령대 정보와 간단한 자기소개를 프로필에 표시하여, 회원 간 매칭 및 예약 시 상대방에 대한 정보를 더 잘 파악할 수 있도록 하기 위함입니다.
- **도메인 객체(VO) 및 TDD 기반 검증 로직 구현**
  - **무엇을**: `AgeGroup` Enum과 `Introduction` Value Object를 신설하고, 내부적으로 50자 제한 검증 로직을 캡슐화했습니다.
  - **왜**: 단순 String 타입 대신 도메인 개념을 명확히 하고 (DDD 원칙), 한줄 소개의 유효성 검증 책임을 객체 스스로 갖게 하여 데이터의 무결성을 보장하기 위함입니다.
- **REST API DTO 및 서비스 로직 연동**
  - **무엇을**: 회원가입(`SignupRequest`), 프로필 수정(`UserUpdateRequest`), 프로필 조회(`ProfileDto`) DTO에 신규 필드들을 반영하고 `AuthService`, `UserService`의 관련 비즈니스 로직을 업데이트했습니다.
  - **왜**: 클라이언트 API 요청/응답 스펙에 맞추어 실제 서비스 로직에서도 해당 정보들을 정상적으로 영속화하고 반환하도록 지원하기 위함입니다.
- **영속성(JPA) 및 서비스 테스트 코드 호환성 수정**
